## Confirmed accurate claims

- README supported formats for vector animation, native 3D models, GLB/PLY sequences, and gaussian splats are accurate at the headline level. README lists `.lottie`, `.riv`, Lottie JSON, OBJ/glTF/GLB, PLY/GLB sequences, and splat-family formats at `README.md:18` and `README.md:59-62`; code classifies OBJ/glTF/GLB only as model formats in `src/stores/timeline/helpers/mediaTypeHelpers.ts:6-12`, detects Lottie JSON by sniffing in `src/stores/timeline/helpers/mediaTypeHelpers.ts:47-54`, imports GLB sequences in `src/stores/mediaStore/helpers/modelSequenceImport.ts:81-177`, and imports splat sequences in `src/stores/mediaStore/helpers/gaussianSplatSequenceImport.ts:103-220`.
- `docs/Features/3D-Layers.md` matches the active shared-scene architecture. The doc says the scene runtime handles planes, primitives, 3D text, OBJ/glTF/GLB, camera clips, and gaussian splats, with no active three.js bridge (`docs/Features/3D-Layers.md:3-9`). Code backs this with the scene layer union `plane | primitive | text3d | model | splat` in `src/engine/scene/types.ts:24-92` and routing from timeline layers to those scene kinds in `src/engine/scene/SceneLayerCollector.ts:78-93` and `src/engine/scene/SceneLayerCollector.ts:141-215`.
- 3D format limits are correctly documented. The docs say FBX is not supported (`docs/Features/3D-Layers.md:217-222`); code only declares `MODEL_EXTENSIONS = ['obj', 'gltf', 'glb']` in `src/stores/timeline/helpers/mediaTypeHelpers.ts:9`.
- glTF/GLB texture claims are supported by code. The 3D doc says base color textures can load from data URIs, external image URIs, or embedded bufferViews (`docs/Features/3D-Layers.md:56-57`); code resolves data URI buffers and texture images in `src/engine/native3d/assets/ModelRuntimeCache.ts:761`, `src/engine/native3d/assets/ModelRuntimeCache.ts:786-846`, and reads material base color textures in `src/engine/native3d/assets/ModelRuntimeCache.ts:837-864`.
- Gaussian splat docs match the current loader and shared-scene path. The doc lists `ply/splat/ksplat/spz/sog/lcc/zip` and SuperSplat-compatible loading (`docs/Features/3D-Layers.md:117-129`); code uses `@playcanvas/splat-transform` and supports `ply`, `splat`, `ksplat`, `spz`, `sog`, and `lcc` in `src/engine/gaussian/loaders/SplatTransformLoader.ts:1-49`, with zip handled through the loader path.
- `docs/Features/Vector-Animation.md` is broadly accurate. It says `.lottie`, sniffed Lottie JSON, and `.riv` import as media (`docs/Features/Vector-Animation.md:9-16`), runtime routing is split through `VectorAnimationRuntimeManager`, Lottie, and Rive managers (`docs/Features/Vector-Animation.md:54-72`), and export is rasterized through the normal GPU path (`docs/Features/Vector-Animation.md:93-112`). Code has the matching service/type split in `src/services/vectorAnimation/VectorAnimationRuntimeManager.ts`, `src/services/vectorAnimation/LottieRuntimeManager.ts`, `src/services/vectorAnimation/RiveRuntimeManager.ts`, and `src/types/vectorAnimation.ts`.
- `docs/Features/Text-Clips.md` is accurate for 2D text and honest about limits. It documents Canvas2D rasterization, area text, stroke/shadow, 50 Google fonts, and no gradient/background/path-editor UI (`docs/Features/Text-Clips.md:5-15`, `docs/Features/Text-Clips.md:53-67`, `docs/Features/Text-Clips.md:69-101`). Code has text canvas rendering in `src/services/textRenderer.ts`, the 50-font catalog in `src/services/googleFontsService.ts:16`, and UI controls in `src/components/panels/TextTab.tsx`.
- Signal IR docs match the implemented slice. `docs/Features/Signal-IR.md` says legacy media still uses established paths while CSV/unknown files become SignalAssets (`docs/Features/Signal-IR.md:11-22`); `UniversalImportOrchestrator` plans concrete signal providers first, then legacy media, then binary fallback in `src/importers/UniversalImportOrchestrator.ts:74-125`. Signal kinds and artifact contracts match `src/signals/types.ts:3-25` and `src/signals/types.ts:86-144`.
- Signal timeline adapters are accurately described. The docs say renderable model/splat artifacts become real model or gaussian-splat clips and everything else becomes a text summary (`docs/Features/Signal-IR.md:53-63`). Code selects model, gaussian splat, then text fallback in `src/runtime/renderers/signalTimelineRendererAdapter.ts:207-232`, materializes stored artifacts from project cache/IndexedDB in `src/runtime/renderers/signalTimelineRendererAdapter.ts:235-280`, and defines the text fallback adapter in `src/runtime/renderers/signalTextRendererAdapter.ts:6-26`.
- Node Workspace docs align with the implemented clip-owned graph model. The doc says it is a dockable selected-clip graph that writes through to existing clip state and does not execute effects itself (`docs/Features/Node-Workspace.md:5-41`). Code imports timeline/dock stores and node graph services in `src/components/panels/nodes/NodeWorkspacePanel.tsx:1-45`, and the context menu exposes AI, Transform, Mask, Color, and Effect Nodes in `src/components/panels/nodes/NodeWorkspacePanel.tsx:1760-1815`.

## Stale or inaccurate claims with code/file evidence

- `README.md` under-describes FlashBoard. It says "AI video and image generation" at `README.md:118` and describes FlashBoard as "text-to-video, image-to-video, and image generation" at `README.md:147`, but current docs/code include audio generation. `docs/Features/FlashBoard.md:5` and `docs/Features/UI-Panels.md:220-233` include ElevenLabs speech and Suno music; `src/services/flashboard/FlashBoardModelCatalog.ts:92-162` declares ElevenLabs/Suno audio catalog entries, and `src/services/flashboard/FlashBoardJobService.ts:480-527` completes ElevenLabs audio jobs as audio files.
- `docs/Features/FlashBoard.md` has a stale Source Map. It lists `src/components/panels/flashboard/FlashBoardWorkspace.tsx`, `FlashBoardToolbar.tsx`, and `FlashBoardCanvas.tsx` at `docs/Features/FlashBoard.md:158-168`, but `rg --files src/components/panels/flashboard` shows only `useFlashBoardRuntime.ts`, `FlashBoardComposer.tsx`, and `FlashBoard.css`. The active tray files are under `src/components/panels/media/MediaAIGenerativeTray.tsx`, `MediaAIGenerativeTrayExpanded.tsx`, and `MediaAIGenerationQueue.tsx`, which are referenced by code at `src/components/panels/media/MediaAIGenerativeTrayExpanded.tsx:8-56`.
- README's architecture tree is stale for the audited systems. It lists `src/services/vectorAnimation` but does not mention `src/engine/scene`, `src/engine/native3d`, `src/engine/gaussian`, `src/signals`, `src/importers`, `src/runtime/wasm`, or `src/runtime/renderers` in the tree (`README.md:309-332`). These are now central to the supported README claims: shared-scene types live in `src/engine/scene/types.ts:24-92`, gaussian loading in `src/engine/gaussian/loaders/SplatTransformLoader.ts:1-49`, Signal IR in `src/signals/types.ts:3-25`, universal import in `src/importers/UniversalImportOrchestrator.ts:74-125`, and Signal timeline rendering in `src/runtime/renderers/signalTimelineRendererAdapter.ts:207-232`.
- README does not surface Signal IR/universal media despite the stated "no unsupported files" ambition in AGENTS and current feature docs. README's supported format table stops at concrete legacy media and splat formats (`README.md:48-70`), while Signal IR docs say unknown files become binary SignalAssets instead of rejected media (`docs/Features/Signal-IR.md:14`, `docs/Features/Signal-IR.md:51`) and code implements that fallback in `src/importers/UniversalImportOrchestrator.ts:111-124`.
- FlashBoard is no longer a standalone visible node canvas in the current implementation. `docs/Features/FlashBoard.md:5-8`, `docs/Features/UI-Panels.md:220-233`, and code all describe the active surface as the Media Panel bottom-right compact composer, but the README feature row still calls it a "Node-based AI canvas" (`README.md:147`). The current mounted surface is `MediaAIGenerativeTray` in the Media Panel (`src/components/panels/MediaPanel.tsx:5265`) with the expanded tray rendering queue plus `FlashBoardComposer` (`src/components/panels/media/MediaAIGenerativeTrayExpanded.tsx:42-56`).
